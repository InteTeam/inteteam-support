# Permanently Deleting a Project

Permanently deleting a project removes it and all its content from the system. This action cannot be undone.

Before a project can be permanently deleted, it must first be archived. This two-step process is intentional — it ensures permanent deletion is a deliberate choice, not an accident.

---

## Who Can Permanently Delete a Project

Only company **Admins** or users with the **Full Access** role can permanently delete a project.

---

## What Permanent Deletion Removes

- The project itself
- All Kanban boards and their columns and cards
- All mindmaps and their nodes
- All project notes
- All project member records

This data cannot be recovered after permanent deletion.

---

## How to Permanently Delete a Project

**Step 1 — Archive the project first**

The project must have **Archived** status before it can be permanently deleted. If it is not already archived, archive it first (see [Archiving a Project](51-archiving-a-project.md)).

**Step 2 — Open the confirmation dialog**

1. Go to **Projects** and select **Archived** from the status filter
2. Locate the project
3. Click the three-dot menu (**⋮**) on the project card
4. Select **Permanently Delete**

**Step 3 — Review the impact**

A dialog appears showing how many boards, mindmaps, and members will be removed. Read this carefully and click **Continue** if you want to proceed.

**Step 4 — Type the project name to confirm**

A second confirmation asks you to type the exact project name to confirm the deletion. This prevents accidental deletions.

1. Type the project name exactly as shown
2. The **Permanently Delete** button becomes active once the name matches
3. Click **Permanently Delete**

The project and all its content are immediately and permanently removed.

---

## Notes

- There is no undo — once permanently deleted, the data is gone
- Moving a project to the trash (the **Move to Trash** action on active projects) is a soft-delete that does not remove content — permanent deletion is separate and requires archived status
- If you only want to hide the project from day-to-day view, use archiving instead of permanent deletion
