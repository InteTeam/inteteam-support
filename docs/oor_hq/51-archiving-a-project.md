# Archiving a Project

Archiving a project removes it from the default **All Active** view without deleting any of its data. Boards, mindmaps, and notes are preserved exactly as they were. You can restore an archived project at any time.

Use archiving for projects that are finished or no longer active but that you want to keep for reference.

---

## Who Can Archive a Project

Only users with the **Admin** or **Full Access** role within your company can archive a project.

---

## What Archiving Does

- Sets the project status to **Archived**
- Hides the project from the default **All Active** view
- All content (boards, mindmaps, notes, members) is kept intact
- Existing members retain their access — they can still open the project by navigating to it directly or by using the **Archived** filter

---

## What Archived Members See

When a project member opens an archived project or one of its boards or mindmaps, a yellow banner appears at the top of the page:

> **This project is archived.** Content is read-only. Contact your admin to restore it.

No edits can be made while a project is archived.

---

## How to Archive a Project

1. Go to **Projects** and locate the project you want to archive
2. Click the three-dot menu (**⋮**) on the project card
3. Select **Archive**
4. A confirmation dialog shows the number of boards, mindmaps, and members affected
5. Click **Archive Project** to confirm

The project disappears from the current view and is moved to the **Archived** status.

---

## Notes

- Archiving is reversible — see [Restoring a Project](52-restoring-a-project.md)
- Archiving is not the same as deleting. The project and all its content remain in the database
- A project cannot be permanently deleted unless it is archived first
